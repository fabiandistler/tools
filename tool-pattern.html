🧬 The Anatomy of an HTML Tool

Every HTML tool follows a four-rule structure. This panel shows the canonical skeleton and explains each rule.

Canonical File Structure

<!-- Single .html file — copy/paste from LLM or GitHub -->
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>My Tool</title>

  <!-- CDN deps here (if truly needed) -->
  <script src="https://cdnjs.cloudflare.com/ajax/libs/marked/9.1.6/marked.min.js"></script>

  <style>
    /* All CSS inline — clean, modern, mobile-friendly */
    :root { --accent: #6c8eff; }
  </style>
</head>
<body>
  <!-- Minimal, functional UI: input → action → output -->

  <script>
    // Vanilla JS only. No React. No JSX. No TypeScript.
    // State via plain objects or localStorage.
    // Errors shown in-UI, never via alert().
  </script>
</body>
</html>

Why "No React"?

JSX requires a build step → you can't copy-paste the output directly. React tools from LLMs have more crashing bugs. Build-step tools are harder to self-host.

Simon's rule: always include "No React" in your LLM prompt. That single phrase avoids a whole rabbit hole.

Why Single File?

• Copy-paste from LLM response directly to GitHub
• Host on GitHub Pages in seconds — no build pipeline
• LLM can rewrite the entire tool from scratch in minutes
• No dependency hell — the file IS the deployable artifact

📋 Copy & Paste Pattern

Clipboard is the most powerful I/O mechanism for HTML tools. Tools can read multiple clipboard formats (text/plain, text/html, images, files) and write back via navigator.clipboard. One-touch copy buttons are essential for mobile.

Key Code

// Read clipboard on paste event (all formats)
document.addEventListener('paste', async (e) => {
  const items = e.clipboardData.items;
  for (const item of items) {
    console.log(item.type); // text/plain, text/html, image/png…
    if (item.type === 'text/plain') {
      item.getAsString(text => console.log(text));
    }
  }
});

// Write to clipboard
await navigator.clipboard.writeText(outputText);

// Rich copy (HTML + plain text simultaneously)
const blob = new Blob([htmlString], { type: 'text/html' });
await navigator.clipboard.write([
  new ClipboardItem({ 'text/html': blob })
]);

🔗 Persist State in the URL

Store tool state in the URL hash — no server required. The state survives page refresh, can be bookmarked, and shared with others. Good for settings that aren't secrets.

Demo: Form State Persists in URL Hash

Edit the fields below — the URL updates automatically. Refresh the page and the values are restored.

Key Code

// Write state to URL hash
function syncToURL() {
  const state = {
    name:  document.getElementById('name').value,
    color: document.getElementById('color').value,
  };
  location.hash = encodeURIComponent(JSON.stringify(state));
}

// Restore state from URL on load
window.addEventListener('DOMContentLoaded', () => {
  if (location.hash) {
    const state = JSON.parse(decodeURIComponent(location.hash.slice(1)));
    document.getElementById('name').value = state.name || '';
  }
});

When to use URL vs localStorage?

URL Hash ✓
Bookmarkable results
Shareable state
Small configuration
Non-sensitive data
localStorage ✓
API keys & secrets
Large content (drafts)
Private per-device data
Long-lived state

💾 localStorage Pattern

Browser localStorage stores data persistently on the user's device without exposing it to any server. Use it for API keys (never hardcode them!), large drafts, and state that shouldn't live in the URL.

API keys are stored in localStorage, never in the HTML source. Use prompt() to collect them.

Key Code

// Auto-save with debounce
let saveTimer;
editor.addEventListener('input', () => {
  clearTimeout(saveTimer);
  saveTimer = setTimeout(() => {
    localStorage.setItem('draft', editor.value);
  }, 800); // save 800ms after last keystroke
});

// Restore on load
editor.value = localStorage.getItem('draft') || '';

// API key pattern — prompt() + localStorage
let apiKey = localStorage.getItem('api_key');
if (!apiKey) {
  apiKey = prompt('Enter your API key:');
  if (apiKey) localStorage.setItem('api_key', apiKey);
}

Key Code

// Read text file
async function handleFile(file) {
  const text = await file.text();       // or file.arrayBuffer() for binary
  console.log(file.name, file.size, file.type);
}

// Read as data URL (for images)
function readAsDataURL(file) {
  return new Promise(resolve => {
    const reader = new FileReader();
    reader.onload = e => resolve(e.target.result);
    reader.readAsDataURL(file);
  });
}

// Drag & drop setup
dropZone.addEventListener('dragover', e => e.preventDefault());
dropZone.addEventListener('drop', e => {
  e.preventDefault();
  handleFile(e.dataTransfer.files[0]);
});

⬇️ File Download Pattern

HTML tools can generate and trigger downloads for any file type — CSV, JSON, SVG, PNG, ICS, plain text — entirely in the browser, no server required.

Demo: Generate & Download Various File Types

Key Code

// Universal download helper
function downloadFile(content, filename, mimeType) {
  const blob = new Blob([content], { type: mimeType });
  const url  = URL.createObjectURL(blob);
  const a = document.createElement('a');
  a.href = url;
  a.download = filename;
  a.click();
  URL.revokeObjectURL(url); // free memory
}

// Examples
downloadFile(csvString, 'export.csv', 'text/csv');
downloadFile(svgMarkup,  'icon.svg',   'image/svg+xml');
downloadFile(jsonString, 'data.json',  'application/json');
downloadFile(icsContent, 'event.ics',  'text/calendar');

// Anthropic Claude via CORS
async function callClaude(prompt, apiKey) {
  const res = await fetch('https://api.anthropic.com/v1/messages', {
    method: 'POST',
    headers: {
      'x-api-key': apiKey,
      'anthropic-version': '2023-06-01',
      'content-type': 'application/json',
      'anthropic-dangerous-direct-browser-access': 'true'
    },
    body: JSON.stringify({
      model: 'claude-haiku-4-5-20251001',
      max_tokens: 1024,
      messages: [{ role: 'user', content: prompt }]
    })
  });
  const data = await res.json();
  return data.content[0].text;
}

// OpenAI equivalent
headers: { 'Authorization': `Bearer ${apiKey}` }
// Gemini: ?key={apiKey} query param

📦 CDN Dependencies

Load third-party libraries from trusted CDNs like cdnjs.cloudflare.com or jsDelivr. Always pin to a specific version for stability. Avoid npm/build pipelines — they kill the "paste and run" workflow.

Recommended Libraries by Use Case

Markdown rendering — marked.js from cdnjs
PDF processing — PDF.js from cdnjs
OCR in browser — Tesseract.js from cdnjs
Code diffs — diff2html or jsdiff
CSV parsing — Papa Parse from cdnjs
Charts — Chart.js from cdnjs
Syntax highlighting — highlight.js from cdnjs
Python in browser — Pyodide + micropip
YAML — js-yaml from cdnjs
QR codes — qrcode.js from cdnjs

CDN URL Pattern

<!-- Pin to a specific version for reproducibility -->
<script src="https://cdnjs.cloudflare.com/ajax/libs/marked/9.1.6/marked.min.js"></script>
<script src="https://cdn.jsdelivr.net/npm/papaparse@5.4.1/papaparse.min.js"></script>

<!-- Pyodide — run Python in browser (via WebAssembly) -->
<script src="https://cdn.jsdelivr.net/pyodide/v0.25.0/full/pyodide.js"></script>
<script>
  async function init() {
    const pyodide = await loadPyodide();
    await pyodide.loadPackage('pandas');
    pyodide.runPython(`import pandas as pd; print("Hello!")`);
  }
</script>

Clipboard Viewer Code (Simon's Most Useful Debug Tool)

document.addEventListener('paste', async e => {
  e.preventDefault();
  console.log('Clipboard items:', e.clipboardData.items.length);
  console.log('Clipboard files:', e.clipboardData.files.length);

  for (const item of e.clipboardData.items) {
    console.log('Type:', item.type, 'Kind:', item.kind);
    if (item.kind === 'string') {
      item.getAsString(s => console.log('Content:', s.substring(0, 200)));
    }
    if (item.kind === 'file') {
      const f = item.getAsFile();
      console.log('File:', f.name, f.size, f.type);
    }
  }
});
